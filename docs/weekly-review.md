# Weekly Review

## Purpose

This document defines the weekly review for FitSculpt.

The weekly review keeps the company aligned, honest, and focused during the beta.

## Outcome

At the end of the weekly review, the founder should know:
- what happened
- what was learned
- what changed
- what matters next
- what should stop, continue, or accelerate

## Inputs

Use:
- Linear status and completed work
- beta feedback received
- release outcomes since the last review
- decision log updates
- memory from the week
- any docs changed in `GitHub /docs`

## Review Questions

### Progress
- What was completed this week?
- What moved but did not finish?
- What stalled?

### Learning
- What did users reveal about usefulness?
- What did users reveal about clarity?
- What did users reveal about completeness?
- What surprised us?

### Product
- What should be fixed now?
- What should be deferred?
- What should be removed from scope?

### Operations
- Did the workflow create friction?
- Did docs stay current enough?
- Did Linear reflect reality?
- Did we preserve important context?

### Decision Quality
- What founder decisions are needed next?
- What was decided this week that should be logged?

## Operating Rules

- Run once per week during beta.
- Keep it lean.
- Base it on evidence, not optimism.
- Distinguish signal from anecdotes when possible.
- Convert decisions and follow-ups into Linear work.
- Update docs when the operating model changes.

## Explicitly Undecided

The exact review day and owner cadence are not fixed yet.
Default assumption: the founder runs or approves the review weekly.
