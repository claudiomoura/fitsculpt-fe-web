# Linear Issue Templates

Use these templates to create manual Linear issue templates for the `Core` team until template automation is explicitly verified and expanded.

## Bug Report

```markdown
## Context
[What is broken, where it shows up, and who is affected]

## Expected Outcome
[What correct behavior should look like]

## Repro / Evidence
- Steps:
- Environment:
- Screenshot / video / logs:

## Severity
[Sev 0 / Sev 1 / Sev 2 / Sev 3]

## Notes
[Constraints, hypotheses, or linked docs]

## Source
Beta Feedback / Founder / Release Review / HQ Chat / Docs
```

## Feedback Follow-Up

```markdown
## Feedback
[User quote, founder observation, or summarized signal]

## Problem
[What confusion, friction, or missing expectation this represents]

## Outcome
[What would improve if this is addressed]

## Scope Notes
[Constraints, links, or affected flows]

## Source
Beta Feedback / Founder / HQ Chat
```

## Decision-Needed Work

```markdown
## Decision Needed
[What founder decision is required]

## Why Now
[Why execution is blocked or why this choice matters now]

## Options
1. [Option one]
2. [Option two]

## Recommendation
[Current recommendation, if any]

## Impact
[What changes depending on the decision]

## Source
Founder / HQ Chat / Docs / Release Review
```
