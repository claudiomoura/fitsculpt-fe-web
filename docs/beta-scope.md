# Beta Scope

## Purpose

This document defines the current beta scope for FitSculpt.

The beta exists to validate whether the product is useful, clear, and complete enough for early users.

## Current Beta Definition

The current beta is:
- free
- 4 weeks
- mobile-first
- distributed as an Android APK
- used first by friends and friends-of-friends

## Beta Goal

The goal of the beta is validation, not scale.

Specifically, the beta is intended to test:
- usefulness
- clarity
- completeness

## In Scope

- mobile-first experience
- Android APK distribution
- direct early-user feedback
- rapid iteration based on observed confusion, friction, and missing pieces
- changes that improve usefulness, clarity, or completeness

## Out Of Scope

Unless explicitly approved by the founder, the beta does not aim to prove:
- large-scale acquisition
- paid conversion
- broad market segmentation
- enterprise readiness
- multi-platform breadth beyond the stated beta context

## Success Lens

The beta is considered directionally successful if it produces credible evidence about:
- whether users find it useful
- whether they understand it clearly
- whether the experience feels complete enough to keep using

## Operating Rules

- Keep scope lean.
- Prioritize learning over polish that does not affect validation.
- Prefer fixes that remove user confusion fast.
- Avoid adding features that do not support beta learning.
- Do not expand scope just because an idea is interesting.

## Required Questions For Any Scope Change

Before expanding scope, answer:
- Does this help validate usefulness, clarity, or completeness?
- Is this required for the Android beta experience?
- Is this reversible?
- What gets deprioritized if this is added?

## Explicitly Undecided

The following are intentionally not defined yet:
- exact success metrics
- target number of beta users
- release frequency inside the 4-week period
- post-beta expansion criteria
